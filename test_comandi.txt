====================================================================
 GUIDA COMPLETA AI TEST MANUALI DEL SERVER IRC (ircserv)
====================================================================

Questa guida ti porta passo passo da zero (compilazione) fino a
testare a mano OGNI comando implementato, con gli errori attesi.

Puoi usare due modi per connetterti:

  A) irssi (client IRC vero e proprio, consigliato: e' gia'
     installato sul sistema) -> vedi comandi mostrati "bene"
     formattati, e' il modo piu' realistico di testare.

  B) nc -C (netcat "raw", manda i comandi grezzi cosi' come li
     scrivi) -> utile per vedere ESATTAMENTE cosa risponde il
     server, incluse le risposte numeriche complete.

     IMPORTANTE: usa SEMPRE "nc -C" e non "nc" senza flag!
     Il flag -C fa in modo che ogni riga finisca con \r\n (CRLF),
     che e' quello che il protocollo IRC richiede. Con "nc" senza
     -C le righe finiscono solo con \n e il server NON le
     processera' mai (restano in attesa di un \r\n che non arriva).


--------------------------------------------------------------------
0) COMPILARE ED AVVIARE IL SERVER
--------------------------------------------------------------------

Apri un terminale nella cartella del progetto:

    cd ~/Desktop/my_irc
    make re

Se tutto va bene vedi "ircserv created!" senza warning/errori.

Avvia il server (porta e password a scelta, qui uso 6667 e
"dontpanic" come esempio):

    ./ircserv 6667 dontpanic

Il server resta in esecuzione stampando log (nuove connessioni,
disconnessioni). Lascialo aperto in questo terminale e apri un
SECONDO terminale per collegarti come client.

Se ti serve fermarlo: Ctrl+C nel terminale del server.

Se riavvii il server subito dopo averlo chiuso e ti da' errore
"Address already in use", aspetta un paio di secondi e riprova
(o cambia porta).


--------------------------------------------------------------------
1) CONNETTERSI E REGISTRARSI (PASS, NICK, USER) -> WELCOME
--------------------------------------------------------------------

IMPORTANTE SULL'ORDINE: manda sempre PASS come primo comando, prima
di NICK e USER (e' quello che fa qualunque client IRC vero). Il
server tollera anche NICK o USER prima di PASS, MA la password deve
comunque arrivare prima che tu abbia finito di mandare ENTRAMBI NICK
e USER: se completi nick+user senza che una PASS corretta sia gia'
arrivata, il server considera la registrazione fallita, risponde
464 e chiude subito la connessione (vedi la sezione 2 per i
dettagli e degli esempi pratici). Quindi la sequenza piu' sicura e'
sempre PASS, poi NICK, poi USER, in quest'ordine.

--- Con irssi ---

In un secondo terminale:

    irssi

Dentro irssi digita (uno alla volta, Invio dopo ognuno):

    /connect 127.0.0.1 6667 dontpanic tuonick

Questo comando da solo manda gia' PASS + NICK + USER internamente.
Se preferisci farlo a mano dentro irssi:

    /server add -network testnet 127.0.0.1 6667 dontpanic
    /connect testnet
    /nick tuonick

Se la registrazione va a buon fine, nella finestra di stato vedrai
i messaggi 001/002/003/004 (il "welcome").

--- Con nc -C (consigliato per vedere i codici numerici) ---

    nc -C 127.0.0.1 6667

Poi scrivi, UNA riga alla volta (Invio dopo ognuna):

    PASS dontpanic
    NICK alice
    USER alice 0 * :Alice Aliceson

Risposta attesa dal server (welcome):

    :ircserv 001 alice :Welcome to the IRC Network alice!alice@localhost
    :ircserv 002 alice :Your host is ircserv, running version 1.0
    :ircserv 003 alice :This server was created today
    :ircserv 004 alice ircserv 1.0 o o

Se vedi questi 4 messaggi, la registrazione funziona. Tieni questa
connessione aperta: la chiameremo "alice" nel resto della guida.

Apri un TERZO terminale e registra un secondo utente allo stesso
modo, ma con nick "bob":

    nc -C 127.0.0.1 6667
    PASS dontpanic
    NICK bob
    USER bob 0 * :Bob Bobson

Da qui in poi useremo "alice" e "bob" per vedere anche i messaggi
che un client manda all'altro (broadcast).


--------------------------------------------------------------------
2) ERRORI DI REGISTRAZIONE DA TESTARE
--------------------------------------------------------------------

Apri una NUOVA connessione (quarto terminale) per ogni test cosi'
non rovini la sessione di alice/bob gia' registrate.

a) Password sbagliata, mandata da sola PRIMA di NICK/USER:

    nc -C 127.0.0.1 6667
    PASS passwordsbagliata

    Atteso: NESSUNA risposta immediata. Come i server IRC veri, la
    password non viene controllata subito: viene solo "ricordata" e
    verificata quando anche NICK e USER sono stati completati. Prova
    ora a completare la registrazione su questa stessa connessione:

    NICK qualcuno
    USER qualcuno 0 * :Qualcuno

    A questo punto (registrazione "tentata" con nick+user pronti ma
    password sbagliata) il server risponde e chiude subito il
    collegamento:

      464 qualcuno :Password incorrect
      ERROR :Closing Link: :Password incorrect

    La connessione si chiude: devi aprirne una nuova per riprovare
    con la password giusta, esattamente come un vero server IRC
    (che manda ERROR + chiude il socket su un fallimento di
    registrazione, non lascia connessioni "a meta'").

a-bis) Stesso identico comportamento se NICK e USER vengono
    completati SENZA aver mai mandato affatto PASS: appena USER (o
    NICK, a seconda di chi arriva per ultimo) completa la coppia
    nick+user, il server verifica se la password e' stata fornita e
    corretta; se non lo e', manda 464 + ERROR e chiude, nello stesso
    identico modo. Provalo su una connessione nuova:

    nc -C 127.0.0.1 6667
    NICK senzapass
    USER senzapass 0 * :Senza Pass

    Atteso: 464 senzapass :Password incorrect, poi la connessione
    si chiude.

    IMPORTANTE - conseguenza pratica: siccome la verifica scatta nel
    momento esatto in cui NICK+USER sono ENTRAMBI completi, se mandi
    PASS DOPO aver gia' mandato sia NICK che USER, la connessione
    verra' gia' chiusa (per mancanza di password valida al momento
    del completamento) PRIMA che la tua PASS possa avere effetto,
    anche se la password che stavi per mandare era quella giusta.
    Quindi l'ordine giusto e' sempre: PASS prima di completare sia
    NICK che USER (va benissimo anche NICK, PASS, USER oppure
    PASS, USER, NICK: l'importante e' che PASS arrivi prima che tu
    finisca di mandare sia NICK che USER). Questo e' esattamente
    come si comportano i client IRC veri (mandano sempre PASS come
    primissimo comando).

b) NICK mancante:

    nc -C 127.0.0.1 6667
    PASS dontpanic
    NICK

    Atteso: 431 :No nickname given

c) NICK non valido (es. inizia con un numero):

    NICK 1abc

    Atteso: 432 1abc :Erroneus nickname

d) NICK troppo lungo (max 9 caratteri):

    NICK abcdefghijk

    Atteso: 432 abcdefghijk :Erroneus nickname

e) NICK gia' in uso (con alice gia' connessa dal passo 1):

    NICK alice

    Atteso: 433 alice :Nickname is already in use

f) Comando mandato PRIMA di essersi registrato (in una connessione
   nuova, senza aver fatto PASS/NICK/USER):

    nc -C 127.0.0.1 6667
    JOIN #test

    Atteso: 451 * :You have not registered

g) PASS mandato DOPO essersi gia' registrati (prova sulla
   connessione di alice, quella gia' registrata):

    PASS dontpanic

    Atteso: 462 alice :You may not reregister

h) Comando sconosciuto (su una connessione gia' registrata, es. bob):

    PIPPO qualcosa

    Atteso: 421 bob PIPPO :Unknown command


--------------------------------------------------------------------
3) PING / PONG
--------------------------------------------------------------------

Sulla connessione di alice:

    PING :ciao

    Atteso: :ircserv PONG ircserv :ciao

Errore (nessun token):

    PING

    Atteso: 409 :No origin specified


--------------------------------------------------------------------
4) JOIN - entrare in un canale
--------------------------------------------------------------------

a) alice crea/entra nel canale #test (chi crea il canale diventa
   automaticamente operatore):

    JOIN #test

    Atteso su alice:
      :alice!alice@localhost JOIN #test
      331 alice #test :No topic is set
      353 alice = #test :@alice          <- la @ indica che e' op
      366 alice #test :End of /NAMES list

b) bob entra nello stesso canale:

    JOIN #test

    Atteso su bob: JOIN + 331/332 + 353 (con "@alice bob") + 366
    Atteso su alice: vedra' arrivare
      :bob!bob@localhost JOIN #test

c) Nome canale non valido (deve iniziare con #):

    JOIN test

    Atteso: 403 test :No such channel

d) JOIN senza parametri:

    JOIN

    Atteso: 461 JOIN :Not enough parameters

e) JOIN multiplo con la virgola (entra in piu' canali insieme):

    JOIN #canaleA,#canaleB

    Atteso: due sequenze JOIN/331/353/366, una per canale.

f) Uscire da TUTTI i canali con "JOIN 0":

    JOIN 0

    (equivale a fare PART su tutti i canali in cui sei)


--------------------------------------------------------------------
5) PART - uscire da un canale
--------------------------------------------------------------------

a) bob esce da #test con un motivo:

    PART #test :mi disconnetto

    Atteso su bob:   :bob!bob@localhost PART #test :mi disconnetto
    Atteso su alice: lo stesso messaggio (broadcast)

b) PART su un canale dove non sei mai entrato:

    PART #canalechenonesiste

    Atteso: 403 #canalechenonesiste :No such channel
    (se il canale esiste ma tu non ci sei dentro: 442 :You're not
    on that channel)

c) PART multiplo con la virgola: come JOIN, es.
   "PART #canaleA,#canaleB :bye"

d) PART senza parametri:

    PART

    Atteso: 461 PART :Not enough parameters

Nota: fai rientrare bob in #test prima di continuare con i test
successivi (JOIN #test), altrimenti alcuni comandi sotto falliranno
con "You're not on that channel" (che e' comunque un test valido,
vedi sezione 9).


--------------------------------------------------------------------
6) PRIVMSG - messaggi privati e di canale
--------------------------------------------------------------------

a) Messaggio al canale (alice scrive, bob deve vederlo su bob):

    PRIVMSG #test :ciao a tutti

    Atteso su bob:   :alice!alice@localhost PRIVMSG #test :ciao a tutti
    Atteso su alice: NIENTE (chi manda non riceve la propria copia,
    e' corretto cosi')

b) Messaggio privato diretto (bob a alice):

    PRIVMSG alice :ciao alice, come va?

    Atteso su alice: :bob!bob@localhost PRIVMSG alice :ciao alice, come va?

c) Nessun destinatario:

    PRIVMSG

    Atteso: 411 :No recipient given (PRIVMSG)

d) Nessun testo (manca la parte con ":" finale):

    PRIVMSG alice

    Atteso: 412 :No text to send

e) Nick inesistente:

    PRIVMSG utenteFantasma :ciao

    Atteso: 401 utenteFantasma :No such nick/channel

f) Canale inesistente:

    PRIVMSG #canaleFantasma :ciao

    Atteso: 401 #canaleFantasma :No such nick/channel

g) Scrivere in un canale in cui NON sei entrato (prova da una
   connessione nuova, non registrata su #test):

    Atteso: 404 #test :Cannot send to channel


--------------------------------------------------------------------
7) TOPIC - vedere/impostare l'argomento del canale
--------------------------------------------------------------------

a) Vedere il topic quando non e' impostato:

    TOPIC #test

    Atteso: 331 #test :No topic is set

b) Impostare un topic (funziona anche se non sei operatore, finche'
   il canale non ha la modalita' +t):

    TOPIC #test :Benvenuti nel canale di test

    Atteso su chi lo manda e su tutti gli altri nel canale:
      :nick!user@host TOPIC #test :Benvenuti nel canale di test

c) Rivedere il topic impostato:

    TOPIC #test

    Atteso: 332 #test :Benvenuti nel canale di test

d) TOPIC su canale dove non sei dentro:

    Atteso: 442 #test :You're not on that channel

e) TOPIC su canale inesistente:

    Atteso: 403 #test :No such channel

Nella prossima sezione vedremo come restringere il TOPIC solo agli
operatori con MODE +t.


--------------------------------------------------------------------
8) MODE - gestione dei canali (operatori, chiave, limite, invite)
--------------------------------------------------------------------

Tutti i MODE che cambiano qualcosa richiedono che TU sia operatore
del canale (alice lo e' perche' ha creato #test).

a) Vedere le mode attuali del canale:

    MODE #test

    Atteso: 324 #test +        (nessuna mode attiva)

b) +t : solo gli operatori possono cambiare il topic

    MODE #test +t

    Atteso (broadcast a tutti nel canale):
      :alice!alice@localhost MODE #test +t

    Verifica: prova ora, da bob (che NON e' operatore), a fare:
      TOPIC #test :provo a cambiare
    Atteso: 482 #test :You're not channel operator

c) +k <chiave> : imposta una password per entrare nel canale

    MODE #test +k segreta

    Poi prova a far entrare un terzo utente (registralo come "carol"
    su una nuova connessione) SENZA la chiave giusta:
      JOIN #test
    Atteso: 475 #test :Cannot join channel (+k)

    Ora con la chiave giusta:
      JOIN #test segreta
    Atteso: entra regolarmente (JOIN + 332/353/366)

    Per togliere la chiave (operatore):
      MODE #test -k

d) +l <numero> : limite massimo di utenti nel canale

    MODE #test +l 2

    Se il canale ha gia' raggiunto il limite, un nuovo JOIN da parte
    di un altro utente da':
      471 #test :Cannot join channel (+l)

    Per togliere il limite:
      MODE #test -l

e) +i : invite-only, si puo' entrare solo se invitati

    MODE #test +i

    Ora un utente NON invitato che prova JOIN #test riceve:
      473 #test :Cannot join channel (+i)

    Vedi la sezione 10 (INVITE) per il flusso completo.

    Per togliere: MODE #test -i

f) +o <nick> / -o <nick> : dare/togliere lo status di operatore

    Da alice (operatore):
      MODE #test +o bob

    Atteso (broadcast): :alice!alice@localhost MODE #test +o bob
    Ora bob e' operatore: puo' fare TOPIC anche con +t attivo, puo'
    fare KICK, puo' cambiare le MODE, ecc. Verificalo facendo, da
    bob:
      TOPIC #test :topic cambiato da bob operatore
    Atteso: funziona (nessun errore 482)

    Per togliere l'op:
      MODE #test -o bob

g) Errori vari da provare:

    - Un utente NON operatore che prova a cambiare una mode:
        (da bob, se non e' piu' operatore) MODE #test +i
        Atteso: 482 #test :You're not channel operator

    - MODE su canale dove non sei dentro:
        Atteso: 442 :You're not on that channel

    - MODE su canale inesistente:
        Atteso: 403 :No such channel

    - Carattere di mode sconosciuto:
        MODE #test +z
        Atteso: 472 z :is unknown mode char to me

    - +o su un nick che non e' nel canale:
        MODE #test +o utenteFantasma
        Atteso: 441 utenteFantasma #test :They aren't on that channel

    - +k o +l senza specificare l'argomento:
        MODE #test +k
        MODE #test +l
        Atteso: 461 MODE :Not enough parameters


--------------------------------------------------------------------
9) KICK - espellere un utente da un canale
--------------------------------------------------------------------

a) alice (operatore) espelle bob da #test:

    KICK #test bob :comportamento scorretto

    Atteso su tutti nel canale (compreso bob):
      :alice!alice@localhost KICK #test bob :comportamento scorretto

    bob risulta rimosso dal canale: se ora bob prova
      TOPIC #test
    ottiene 442 :You're not on that channel (perche' non e' piu' dentro)

b) KICK fatto da chi NON e' operatore:

    (fai rientrare bob con JOIN #test, poi prova, da bob):
      KICK #test alice :provo io

    Atteso: 482 #test :You're not channel operator

c) KICK verso un utente non presente nel canale:

    KICK #test utenteFantasma :via

    Atteso: 441 utenteFantasma #test :They aren't on that channel

d) KICK su canale dove tu stesso non sei:

    Atteso: 442 :You're not on that channel

e) KICK con parametri mancanti:

    KICK #test

    Atteso: 461 KICK :Not enough parameters


--------------------------------------------------------------------
10) INVITE - invitare un utente in un canale (utile con +i)
--------------------------------------------------------------------

Scenario completo (riusa alice come operatore di #test, e registra
un nuovo utente "carol" su una connessione pulita):

    nc -C 127.0.0.1 6667
    PASS dontpanic
    NICK carol
    USER carol 0 * :Carol C

a) Attiva invite-only sul canale (da alice):

    MODE #test +i

b) carol prova ad entrare senza invito:

    JOIN #test

    Atteso: 473 #test :Cannot join channel (+i)

c) alice invita carol:

    INVITE carol #test

    Atteso su alice: 341 carol #test
    Atteso su carol: :alice!alice@localhost INVITE carol :#test

d) Ora carol puo' entrare:

    JOIN #test

    Atteso: entra regolarmente.

e) Errori da provare:

    - INVITE verso un nick inesistente:
        INVITE utenteFantasma #test
        Atteso: 401 utenteFantasma :No such nick/channel

    - INVITE su un canale dove tu (chi invita) non sei dentro:
        Atteso: 442 :You're not on that channel

    - INVITE fatto da un non-operatore MENTRE il canale e' +i:
        Atteso: 482 :You're not channel operator
        (se il canale NON e' +i, invece, chiunque nel canale puo'
        invitare: e' corretto per specifica)

    - INVITE verso qualcuno gia' presente nel canale:
        Atteso: 443 nick #test :is already on channel

    - INVITE con parametri mancanti:
        INVITE
        Atteso: 461 INVITE :Not enough parameters


--------------------------------------------------------------------
11) NICK - cambiare nickname a registrazione avvenuta
--------------------------------------------------------------------

Da una connessione gia' registrata (es. carol):

    NICK carolina

    Atteso: carol stessa vede
      :carol!carol@localhost NICK :carolina
    E chiunque condivida un canale con lei vede lo stesso messaggio
    (per notificare il cambio nick a chi la conosce).

    Riprova gli errori della sezione 2 (nick doppio, nick invalido,
    nick troppo lungo) anche qui, sono identici dopo la registrazione.


--------------------------------------------------------------------
12) QUIT - disconnettersi
--------------------------------------------------------------------

Da una connessione (es. bob):

    QUIT :a dopo!

    Atteso su bob: ERROR :Closing Link   (poi la connessione si chiude)
    Atteso su chi condivide un canale con bob (es. alice, se sono
    ancora insieme in #test):
      :bob!bob@localhost QUIT :a dopo!

    Dopo QUIT il nickname "bob" torna libero: puoi riconnetterti e
    registrarti di nuovo con NICK bob su una nuova connessione.


--------------------------------------------------------------------
13) TEST "REALISTICO" CONSIGLIATO (ordine suggerito con 3 terminali)
--------------------------------------------------------------------

Terminale 1: ./ircserv 6667 dontpanic          (lascialo aperto)

Terminale 2 (alice): nc -C 127.0.0.1 6667
    PASS dontpanic
    NICK alice
    USER alice 0 * :Alice
    JOIN #test
    MODE #test +t
    TOPIC #test :canale di prova

Terminale 3 (bob): nc -C 127.0.0.1 6667
    PASS dontpanic
    NICK bob
    USER bob 0 * :Bob
    JOIN #test
    PRIVMSG #test :ciao a tutti
    TOPIC #test :provo a cambiarlo   <- deve fallire (482, non e' op)

Torna su alice:
    MODE #test +o bob
    KICK #test bob :test kick
    QUIT :fine test

Su bob (dopo il kick, prima del quit di alice) prova a rientrare:
    JOIN #test

Se tutte le risposte numeriche corrispondono a quelle descritte
sopra in ogni sezione, i comandi funzionano correttamente.


--------------------------------------------------------------------
NOTE FINALI
--------------------------------------------------------------------

- Ricorda: con "nc -C" ogni riga che scrivi e premi Invio e' un
  comando IRC separato: scrivine uno alla volta e aspetta la
  risposta prima del successivo, altrimenti diventa difficile
  leggere cosa risponde a cosa.

- Se una connessione "si blocca" senza risposta, quasi sempre e'
  perche' hai dimenticato il flag -C di nc (niente CRLF -> il
  server aspetta ancora il resto della riga).

- Il server logga sul suo terminale (Terminale 1) ogni nuova
  connessione e disconnessione: utile per capire se un client si e'
  davvero collegato/disconnesso.
